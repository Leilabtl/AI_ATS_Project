# HR Compass — AI Talent Navigation

An advanced, explainable AI-powered Applicant Tracking System (ATS) that combines keyword-based
pre-screening with GPT deep analysis to rank candidates, detect bias, and generate professional
recruitment reports.

## Key Features

| Feature | Description |
|---|---|
| **Two-phase AI pipeline** | Phase 1: fast keyword pre-screening · Phase 2: GPT chain-of-thought deep analysis |
| **Explainable scoring** | 6-factor weighted formula with per-factor breakdown |
| **Bias detection** | Flags name-based, age-related, and gender-coded language |
| **GPT executive summary** | Candidate-specific narrative generated by `gpt-4o-mini` |
| **Skill gap roadmaps** | Prioritised learning paths with time estimates |
| **Interview prep** | Auto-generated focus areas per candidate |
| **PDF & CSV reports** | One-click professional report generation |
| **Email templates** | BCC-ready shortlist / longlist / rejection templates |
| **Candidate pool** | Cross-job persistent storage with cross-role recommendations |
| **Batch processing** | 50–500+ CVs at ~1–2 s each |

### Prompt Engineering Decisions

- **Chain-of-thought** (`PROMPT_VERSION = "3.0"`): the system prompt instructs GPT to reason
  step-by-step through requirements before producing JSON — reduces hallucinations.
- **JSON mode** (`response_format={"type":"json_object"}`): guarantees parseable output.
- **Dynamic context injection**: pre-computed matched/missing skills and similarity scores
  are injected into the user prompt so GPT can focus on nuanced judgement, not counting.
- **Token-aware truncation**: CV capped at 4 000 chars, JD at 2 000 chars to stay within budget.
- **Output schema validation + retry**: response validated against a strict field/type schema;
  retried once automatically on failure.
- **Prompt injection protection**: CV text sanitised before sending — known jailbreak patterns
  replaced with `[REDACTED]`.
- **Cost monitoring**: token usage tracked per session; estimated cost displayed in Settings tab.

## Local Setup

### 1. Clone the repository

```bash
git clone https://github.com/Leilabtl/AI_ATS_Project.git
cd AI_ATS_Project
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Configure environment variables

```bash
cp .env.example .env
# Open .env and paste your OpenAI API key
```

### 4. Run the app

```bash
streamlit run streamlit_app.py
# App opens at http://localhost:8501
```

### 5. Run tests

```bash
python -m pytest tests/ -v
```

## Deployment

### Streamlit Community Cloud (recommended — free)

1. Push code to your GitHub `main` branch.
2. Go to [share.streamlit.io](https://share.streamlit.io) → **New app**.
3. Select your repo, branch `main`, main file `streamlit_app.py`.
4. Under **Advanced settings → Secrets**, add:
   ```
   OPENAI_API_KEY = "sk-..."
   ```
5. Click **Deploy** — the app gets a public `*.streamlit.app` URL.

> **Important:** never commit your `.env` file. It is already listed in `.gitignore`.

| Measure | Implementation |

## Scoring Formula

```
Final Score =
  0.35 × Semantic Similarity (Jaccard word overlap)
+ 0.25 × Skills Match       (JD skills found in CV)
+ 0.15 × Experience Relevance
+ 0.15 × Keyword Density
+ 0.05 × Culture Fit
+ 0.05 × Seniority Alignment
```

The formula is fully transparent and displayed per-candidate in the UI.

## Known Limitations & Future Work

- **Semantic similarity** uses Jaccard (word overlap), not neural embeddings — synonyms are not matched. A future version could use `sentence-transformers` for true semantic search.
- **GPT cost** scales linearly with candidate count. For very large batches (500+ CVs), a two-stage pipeline (fast filter then deep analysis for top 50) would reduce cost significantly.
- **PDF parsing** may fail on scanned / image-only PDFs. Adding OCR (e.g. Tesseract) would handle these.
- **Single-user** — the persistent `candidate_pool.json` is shared. A database backend (SQLite or PostgreSQL) would be needed for multi-user deployment.
